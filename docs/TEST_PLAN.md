# Usability Test Plan: CSU IS Career Hub

## 1. Test Overview
* **Goal:** Validate the "Happy Path" for student internship discovery and application.
* **Methodology:** Unmoderated remote testing (Figma Prototype).
* **Target Audience:** 5 CSU Information Systems Students (Junior/Senior standing).
* **Success Metric:** 100% completion rate on "Task 2" (Finding the Sherwin-Williams application).

## 2. Test Script & Scenarios

### **Introduction (Read to Participant)**
"Hi, we are designing a dedicated internship portal for the IS Department. Iâ€™m going to show you a prototype and ask you to complete three specific tasks. Please speak your thoughts aloud as you navigate. This is a test of the design, not of you."

### **Task 1: First Impressions (5 Seconds)**
* **Prompt:** "Look at this homepage. Without clicking anything, who do you think this website is for?"
* **Success Criteria:** User identifies "Monte Ahuja" or "IS Students" immediately.
* **Pass / Fail:** [ ]

### **Task 2: The Search Flow**
* **Prompt:** "You want to find an Information Systems internship specifically at **Sherwin-Williams**. Show me how you would find it."
* **Success Criteria:**
    1.  User clicks "Browse Internships" or "Find IS Internships".
    2.  User identifies the Sherwin-Williams card.
    3.  User clicks the card to view details.
* **Pass / Fail:** [ ]

### **Task 3: The Application Action**
* **Prompt:** "You've read the job description and decided to apply. What do you do next?"
* **Success Criteria:** User successfully locates and clicks the green "Apply Now" button.
* **Pass / Fail:** [ ]

## 3. Post-Test Questionnaire
1.  Did the job locations (Cleveland/Mayfield) make the site feel more relevant to you than LinkedIn?
2.  Was there any information missing from the Job Detail card that you would need before applying?
3.  On a scale of 1-5, how confident did you feel that these jobs were curated for your major?
